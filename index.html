大家猜猜看...

📝 Google Form 會自己寄信嗎？ ➔ 不會。
📊 Google Sheet 會自己通知主管嗎？ ➔ 不會。

所以，中間是不是需要一個人，
負責叫大家起床工作？

🧠

GAS 主角登場：
Google 的總務主任 (大腦)

部門之間平常是不講話的。
這時，我們需要一個 大腦 來指揮大家：

📝 有人填表單
➔
🧠 GAS (大腦)
收到通知
➔
📊 叫 Sheet 存檔
➔
✉️ 叫 Gmail 寄信

GAS 不負責打雜，它負責「把工具串起來工作」！

⏰

這個大腦什麼時候「起床工作」？

這叫做「觸發時機」 (Trigger)

  • 📨
    有人送出表單時，起床工作
    (GAS Trigger 名稱：onFormSubmit)
  • 📊
    有人修改表格時，起床工作
    (GAS Trigger 名稱：onEdit)
  • ⏰
    每天下午五點時，起床工作
    (GAS Trigger 名稱：Time-driven)
🤖

總結：你講流程 ➔ AI 翻譯 ➔ GAS 執行。

Q1：哪裡一直重複？

每次都在問家屬一樣的基本資料。

用 Google Form 統一入口

🧠 觸發時機：表單一送出，GAS 就醒來接手處理

(💡 現場 Demo：填寫意見反映表單送出)

Q2：資料散在哪裡？

LINE、紙本、Email、電話，找資料像考古。

讓 GAS 自動把資料收進 Sheet

以前是痛苦「找訊息」，現在是輕鬆「查案件」。

(💡 現場 Demo：展示 Sheet 欄位自動匯入)

Q3：有沒有固定的判斷？

交給 GAS 自動抓關鍵字亮燈

「肯定、謝謝」 ➔ 🟢 綠燈
「提告、受傷」 ➔ 🔴 紅燈
「其他建議」   ➔ 🟡 黃燈
➔
// 🧠 GAS 按照你的規則判斷
如果 (看到肯定內容) {
  亮綠燈;
} 否則如果 (看到高風險) {
  亮紅燈;
}

(💡 放心，這不是要大家手寫！
你以後看到 AI 產出的程式，大概就是長這樣。)

Q4：發生事情時，要通知誰？

🟢 綠燈
主管
(收到好評)
🟡 黃燈
主管 + 社工 + 書記
(同步知悉)
🔴 紅燈
主管 + 社工 + 管理層
(危機通報)

「工具上是寄信，
管理上是把現場分工變成自動通知規則。」

🧠 觸發時機：GAS 判斷完燈號後，自動叫 Gmail 寄信

Q5：收到以後，怎麼追蹤進度？

  • 進度分三階：⏳ 待處理 ➔ 🚧 處理中 ➔ ✅ 已結案
  • 自動封存：結案後 GAS 自動幫你收進倉庫。
  • 誤改救援：不小心改錯，還能搬回主表。
  • 手動變色：你手動改燈號，GAS 自動幫你換底色。
🧠 觸發時機：有人修改表格，GAS 就發現並自動整理

(💡 現場 Demo：手動改結案、觸發自動搬移)

Q6：主管最後想看什麼？

主管不想看 100 筆明細，只想知道：

紅燈幾件？待處理幾件？誰上榮譽榜？誰在警戒榜？

戰情室 ＋ 每天自動發送日報

🧠 觸發時機：下午五點一到，GAS 自動產出報表並寄出

(💡 現場 Demo：展示 Dashboard 與日報信件)

🕸️

所以，GAS 到底可以幫你做什麼？

GAS 就是 Google Workspace 的 流程引擎

📥 1. 接收訊號 (Trigger)
📝 填寫表單
💬 收到 LINE
➔
🧠
GAS 大腦
➔
📤 2. 派發任務 (Action)
📊 更新表格
✉️ 自動寄信
📄 產出文件
📁 歸檔雲端

只要把兩邊連起來，中間的苦工 GAS 全包了！

我們不是從寫程式開始，
而是從梳理流程開始。

  • 📝 重複的事 ➔ 表單化
  • 🗂️ 分散的資料 ➔ 表格化
  • 🧠 固定的判斷 ➔ 規則化
  • 📨 固定的通知 ➔ 自動化
  • 📊 主管看結果 ➔ 報表化
🎯

今天不是學程式，
是學怎麼讓 Google 幫你工作！

  1. Google 本來就有許多免費工具。
  2. GAS 是大腦，負責把工具串聯起來。
  3. 你只要講清楚流程，AI 會幫你翻譯，
    讓 Google 自己完成工作！

(👇 等等你會發現，我們今天沒有寫多少程式，反而大部分時間都在設計流程！)
